## **프로그램** 과 **프로래밍언어** 이란?

- 프로그램 (program)

> **프로그램**(program) 은 **컴퓨터에서 특정 목적을 수행하기 위해 프로그래밍 언어로 작성된 소스를 기계어로 번역 한 것**을 의미합니다.

<br>

- 기계어

> 기계어는 0과 1로 이루어진 이진코드 입니다.
> 기계어는 사람이 이해하기 어려운 언어 입니다.

<br>

- 프로그래밍언어 (program language)

> 프로그래밍 언어는 사람의 언어와 기계어의 다리 역할을 합니다.
>
> 프로그래밍 언어는 '고급언어'에 속하며, 사람이 이해할 수 있는 언어입니다. 반면에 컴퓨터가 이해하지 못하는 언어입니다.
>
> '저급언어'는 기계어에 가까운 어어이며, 컴퓨터가 이해할 수 있지만 사람이 이해하기 어려운 언어를 의미합니다.
>
> 프로그래밍 언어는 컴파일(compile) 과정을 통해서 컴퓨터가 이해할 수 있는 기계어로 변환 시켜 컴퓨터가 해당 언어에 대한 처리를 하도록 합니다.


<br><br>

## Java의 특징

- (특징1) 이식성이 높은 언어 입니다.

> **이식성** : 서로 다른 실행 환경을 가진 시스템간에 프로그램을 옮겨 실행할 수 있는 것을 의미합니다.


> 자바언어로 개발된 프로그램은 소스파일을 다시 수정하지 않아도, **자바 실행환경**(JRE: Java Runtime Environment)**이 설치되어있는 모든 운영체제에서 실행이 가능**합니다.


<br>

- (특징2) 객체지향 언어 입니다.

> 자바는 100% 객체지향 언어 입니다.

> 객체지향언어의 특징은 '캡슐화', '상속', '다형성' 기능이며, 자바는 이 기능들을 지원하고 있습니다.

> 객체지향프로그래밍에 사용되는 언어를 객체지향 언어 라고 합니다.
> **[객체지향프로그래밍]**은 밑에 설명해놓겠습니다.

<br>

- (특징3) 함수적 스타일 코딩을 지원합니다.

> 함수적프로그래밍은 대용량 데이터의 병렬처리과 이벤트 지향프로그래밍에 사용됩니다.
>
> 자바8부터 람다식(Lambda Expression)이라는 함수적프로그래밍을 지원하고, 람다식을 사용하면 컬렉션의 요소를 필터링, 매핑, 집계처리가 쉬워지고 코드가 간결해집니다. 

<br>

- (특징4) 메모리를 자동으로 관리합니다.

> 메모리는 자바가 직접 관리를 하며, 개발자가 직접 메모리에 접근할 수 없도록 설계되어있습니다.
>
> 객체 생성 시 자동적으로 메모리 영역을 찾아서 할당하고, 객체 사용이 완료되면 쓰레기 수집기(Garbage Collector)를 실행시켜 자동적으로 사용하지 않은 객체를 제거시켜줍니다.
> 
> 개발자는 메모리 관리의 수고스러움을 덜고, 핵심 기능 코드 작성에 집중할 수 있습니다.

<br>

- (특징5) 다양한 애플리케이션을 개발할 수 있습니다.

> 자바는 다양한 운영체제(OS: Operating System)에서 실행되는 프로그램을 개발할 수 있습니다.

> 자바는 특히 웹애플리케이션, 모바일 앱 등 프로그램을 개발할 수 있습니다.

<br>

- (특징6) 멀티스레드(Multi-Thread)를 쉽게 구현할 수 있습니다.

> 멀티스레드 프로그래밍(Multi Thread Programming) 의 목적은
>> *하나의 프로그램이 동시에 여러가지 작업을 처리*해야 할 경우와 *대용량 작업을 빨리 처리하기 위해 서브작업으로 분리해서 병렬처리* 하기 위해서 입니다.


> 자바는 스레드 생성 및 제어와 관련된 라이브러리 API를 제공하기 때문에 실행되는 운영체제에 상관없이 멀티스레드를 쉽게 구현할 수 있습니다. 

<br>


- (특징7) 동적로딩(Dynamic Loading)을 지원합니다.

> 자바애플리케이션은 여러 개의 객체가 연결되어 실행 됩니다. 이 객체들은 클래스로 부터 생성됩니다.
> 
> 애플리케이션이 실행될 때 모든 객체가 생성되지 않고, *객체가 필요한 시점에 클래스를 동적로딩해서 객체를 생성*합니다.
> 
> 개발 완료 후 유지보수(수정)가 발생하더라도, *해당 클래스만 수정하면 되므로 전체 애플리케이션을 다시 컴파일할 필요가 없습니다.* 

<br>

- (특징8) 오픈소스 라이브러리가 풍부합니다.

## **객체지향 프로그래밍** 이란?

> **객체지향 프로그래밍**(OOP: Object Oriented Program) 은 프로그램 개발하는 기법에 해당되며, **부품에 해당하는 객체들을 먼저 만들고 이것들을 하나씩 조립 및 연결해서 전체프로그램을 완성하는 기법**을 의미합니다.

<br><br>

## **객체** 란?

> 객체(Object)란 물리적으로 존재하거나 추상적으로 생각할 수 있는 것 중 **자신의 속성을 가지고 있고 다른 것과 식별 가능한 것**을 의미합니다.
> 
> 객체는 **속성** 과 **동작**으로 구성되어 있습니다.
>
> 자바는 이 **속성**과 **동작**들을 각각 **필드(field)** 와 **메소드(method)** 라고 부릅니다.
>
> 객체 모델링(Object Modeling) 은 현실 세계의 객체를 소프트웨어 객체로 설계하는 것을 의미하며, 속성과 동작을 추려내어 소프트웨어 객체의 필드와 메소드로 정의하는 과정이라고 볼 수 있습니다.

## JVM (Java Virtual Machine)

> 자바를 실행키는 가상의 기계
>
> *자바프로그램*은 완전한 기계어가 아닌, 중간 단계의 *바이트코드* 입니다. 
>
> JVM 은 **바이트코드를 해석하고 자바프로그램을 실행시키는 가상의 운영체제 역할**을 합니다.
>
> 자바프로그램이 **여러 운영체제에서 동일한 실행결과**가 나오게 하기 위해서 JVM은 운영체제와 자바프로그램 을 중계합니다.
> 
> **바이트 코드는 모든 JVM에서 동일한 실행결과를 보장**하지만, JVM은 **운영체제에 종속적** 입니다.
>
> 자바 프로그램을 운영체제가 이해하는 기계어로 번역해서 실행해야되기 때문에 JVM은 운영체제에 맞게 설치돼야합니다.
> 
> JVM은 JDK와 JRE를 설치하면 자동으로 설치되는데, JDK와 JRE가 운영체제별로 제공됩니다.

<br>

![](https://miro.medium.com/max/1400/1*slIuYO633BCuBh_gfYRmGg.png)


> 소스파일(확장자가 `*.java` 인 파일) 을 컴파일러(`javac.exe`)로 컴파일하면 확장자가 바이트코드 파일(확장자가 `*.class`) 이 생성됩니다.
> >
> 바이트 코드 파일은 JVM 구동명령어(`java.exe`)에 의해 JVM에서 해석되고 해당 운영체제에 맞게 기계어로 번역됩니다.
>
> 바이트코드는 하나지만, JVM에 의해서 번역되는 기계어는 운영체제에 따라 다릅니다.

<br><br>


## JDK (Java Development Kit) 란?

- JDK: JRE + 개발에 필요한 도구

> 프로그램 개발에 필요한 JVM, 라이브러리 API, 컴파일러 등 개발 도구가 포함되어 있습니다. 
>
> JDK 내부의 bin 디렉토리는 컴파일러인 `javac.exe` 와 JVM 구동명령어인 `java.exe` 가 포함되어 있습니다.


<br><br>

## JRE (Java Runtime Environment) 란?

- JRE: JVM + 표준 클래스 라이브러리

> 프로그램 실행에 필요한 JVM, 라이브러리 API 만 포함되어 있습니다.
